# Airbnb Clone Backend — User Stories

## Overview

User stories are short, simple descriptions of a feature told from the perspective of the person who desires the new capability. They are derived from the use case diagram and reflect the real-world interactions users will have with the system.

---

## How to Use This Document

- **Developers**: Use these stories to define API endpoints, database operations, and access control logic.
- **Product Managers**: Prioritize features and plan sprints based on user needs.
- **QA Engineers**: Design test cases that validate each user story.
