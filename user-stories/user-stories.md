## Objective

Translate use case diagram interactions into actionable user stories that guide backend development and feature implementation.

## User Stories

### 1. User Registration
_As a user, I want to be able to register an account so that I can book properties or list my own._

### 2. Property Booking
_As a guest, I want to book a property for specific dates so that I can plan my stay._

### 3. Listing Management
_As a host, I want to add, edit, and delete property listings so that I can manage my rental offerings._

### 4. Payment Processing
_As a guest, I want to securely pay for my booking so that I can confirm my reservation._

### 5. Review Submission
_As a guest, I want to leave a review after my stay so that I can share feedback with other users._

### 6. Messaging
_As a user, I want to send and receive messages so that I can communicate with hosts or guests._

### 7. Admin Oversight
_As an admin, I want to monitor users, listings, and bookings so that I can ensure platform integrity._

---

## Notes

- These stories are derived from the use case diagram and reflect core backend interactions.
- Each story can be expanded into tasks, API endpoints, and database operations.
- Additional stories can be added for edge cases, error handling, and role-specific permissions.
